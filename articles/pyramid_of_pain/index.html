What is the Pyramid of Pain and why is it needed?

Pyramid of pain is a concept that helps security teams assess the effectiveness of their threat intelligence program and helps to identify ways to improve them. Essentially, this concept explains the value of indicators of compromise and how they can potentially agitate adversaries. Moreover, it explains how the loss of some indicators of compromise can cause more harm to adversaries than the loss of others.

HASH VALUES:

What is a Hash Value?

According to Microsoft, a hash value is a numeric value of a fixed length that uniquely identifies data. In other words, it is a condensed representation of its input that is produced by a cryptographic primitive called a hash function/hash algorithm. These cryptographic hash functions are one-way functions that are extremely difficult to reverse. A couple of common hashing algorithms are: MD5, SHA-1, SHA-2.

Why is this Important?

This is important information because security professionals usually take malicious software and run it through a hashing program that produces a unique hash for that specific malicious software. There are open source tools like VirusTotal and Metadefender Cloud - OPSWAT that allow security professionals to do lookups on these unique hashes to find a malicious file. Although these hash values make it easy for us to spot malicious files that we have hashes for, an attacker can modify the malicious software/file by a single bit, which would produce a completely different hash value, making it trivial for security professionals to identify the malicious file.

IP ADDRESS:

What is an IP address?

An IP address is a unique address that identifies any device connected to a network. IP addresses allow for computers to communicate with each other both over the network as well as over the internet. Many organizations use defense tactics, such as firewalls, to block, drop, or deny inbound requests from specific IP addresses. Although this tactic may make it slightly difficult for attackers, it is not bulletproof. Experienced attackers can simply use a new public IP address or something like Fast Flux to make it more difficult for security professionals to carry out IP blocking. Fast Flux, in simple terms, is a DNS technique that has multiple IP addresses associated with a domain name and is constantly changing, making it harder for firewalls to block their IP addresses.

DNS: Domain names:

Domain names are IP addresses that are put into a string of text. For example: Google.com = 8.8.8.8 and/or 8.8.4.4. Many times attackers use a technique called typosquatting. Typosquatting is creating domain names that are similar to the real domains. An example of this is the following: ggoogle.com instead of google.com. The reason they do this is because they are hoping that you misspell the domain name and end up getting directed to their malicious page. Many attackers also use something called a Punnycode which uses Unicode characters in the domain name to imitate the known domain name. It is quite annoying for attackers to change their domain because they would most likely need to buy a new domain, as well as register it and modify DNS records.

Network/HOST ARTIFACTS:

These include traces of the attacker such as registry values, suspicious process execution, and values known to be created by specific pieces of malware, files, or directories. On this level, the attacker will feel annoyed as it requires the attacker to change his attack tools and procedures.

TOOLS:

At this point, attackers have exhausted their efforts trying to break into our system or they attempt to create new tools that serve the same purpose. However, doing so would cost them more money as they would need to build these new tools. Tools are malicious software or utilities used by the attackers to perform their attacks. Some examples of tools that attackers may use include vulnerability scanners, malicious documents, brute force attacks to steal credentials, and etc. At this stage, security professionals may use great weapons to defend themselves against attackers; some of these include antivirus signatures, detection rules, and YARA rules.

TTPS:

If security professionals can detect and respond to Tactics, Techniques, and Procedures (TTPs) quickly, it will leave the attacker in the dust. If we are able to detect and respond to their ways, then we can stop these attackers from breaching our systems.
